## 개인화 추천 알고리즘

```
User-Item Matrix:
  - 행(row): 사용자 ID
  - 열(col): 결함 ID / TC ID
  - 값: 상호작용 가중치
      - 분석 시 유사 결함 클릭: 1.0
      - TC 편집: 2.0
      - TC 코멘트: 1.5
      - 분석 완료 (해당 결함 영향): 0.5

협업 필터링 (User-Based CF):
  1. 현재 사용자와 코사인 유사도가 높은 Top-5 유사 사용자 탐색
  2. 유사 사용자가 높게 평가한 결함/TC 중 현재 사용자 미경험 항목 추출
  3. Top-5 추천 반환

콜드 스타트 (상호작용 < 5건):
  → 전체 사용자 기준 인기 순 (상호작용 합계) Top-5 반환
```

## PDF 보고서 구성

```
[QA 영향도 분석 보고서]
─────────────────────────────
1. 요약 (Executive Summary)
   - 변경 내용
   - 영향도 점수 + 등급 (Low/Medium/High/Critical)
   - 주요 영향 모듈

2. 유사 결함 목록 (Top-5)
   - 결함 제목, 심각도, 유사도 점수, 발생일

3. 영향도 상세 분석
   - 영향 영역 목록
   - 잠재적 Side Effect

4. 제안된 테스트케이스
   - 각 TC: 제목, 절차, 기대 결과

5. 분석 정보
   - 분석 일시, 분석자, 시스템 버전
─────────────────────────────
```
